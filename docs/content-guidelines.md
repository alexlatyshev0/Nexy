# Content Guidelines

> Правила написания контента для сцен: описания, промпты, заголовки.

---

## user_description

**Назначение:** Фактическое описание сцены для пользователя.

### Главное правило

**Описывай ЧТО ПРОИСХОДИТ, а не ЧТО ЧУВСТВУЮТ.**

Пользователь сам решит, нравится ему это или нет. Не навязывай эмоции.

### Правила

| DO | DON'T |
|----|-------|
| Факты: что происходит | Эмоции: что чувствуют |
| Нейтральный тон | Пафос, драматизация |
| Конкретные действия | Абстрактные описания |
| Столько деталей, сколько нужно | Искусственные ограничения |

### Примеры

**Плохо:**
- "Они с трепетом познают тела друг друга, преодолевая страх и стеснение" — навязывает эмоции
- "Момент невероятной близости и доверия" — пафос
- "Она отдаётся ему полностью, растворяясь в ощущениях" — драматизация
- "Секс втроём" — слишком мало деталей для сложной сцены

**Хорошо (простые сцены):**
- "Пара занимается сексом первый раз"
- "Он связывает ей руки шёлковой лентой"
- "Она шлёпает его по заднице"

**Хорошо (сложные сцены):**
- "Два мужчины занимаются сексом с одной женщиной. Один спереди держит её за горло. Второй сзади входит в её попку и шлёпает её."
- "Она на коленях перед ним в ошейнике. Он держит поводок и направляет её голову к своему члену."
- "Он привязан к кровати. Она сидит на его лице, пока её подруга делает ему минет."

### Формула

```
[КТО] + [ЧТО ДЕЛАЕТ] + [детали по необходимости]
```

Для сложных сцен — описывай каждое действие/позицию участников.

## ai_description

**Назначение:** Технический контекст для AI-системы (matching, recommendations).

### Правила

- Описывает суть сцены для алгоритма
- Может содержать термины, теги
- Объясняет что тестирует эта сцена
- Не показывается пользователю напрямую

### Примеры

```json
{
  "ru": "Базовая ориентация: доминирование, подчинение или переключение. Определяет power dynamic для всех последующих сцен.",
  "en": "Basic orientation: dominance, submission or switching. Sets power dynamic for all subsequent scenes."
}
```

```json
{
  "ru": "Тест на интерес к групповому сексу. MMF конфигурация. Связано с: cuckolding, hotwife, bi-curious.",
  "en": "Testing interest in group sex. MMF configuration. Related: cuckolding, hotwife, bi-curious."
}
```

## image_prompt

**Назначение:** Промпт для генерации изображения сцены.

### Запрещённые слова

НИКОГДА не использовать (стиль задаётся отдельно):

```
photorealistic, realistic, photo, photography
masterpiece, best quality, high quality, detailed
4k, 8k, uhd, hd, high resolution
sharp, professional, cinematic, artistic
beautiful, stunning, amazing, perfect
natural lighting, studio lighting, dramatic lighting
```

### Структура промпта

```
[КТО: man/woman явно] + [ЧТО ДЕЛАЕТ: поза, действие] + [ГДЕ: место] + [ДЕТАЛИ: предметы, одежда]
```

### Примеры

**Плохо:**
- "Beautiful couple in passionate embrace, high quality, 4k" — запрещённые слова
- "Two people kissing" — не указан пол
- "Intimate moment" — слишком абстрактно

**Хорошо:**
- "man and woman kissing on bed, his hands in her hair, bedroom with dim lamp light"
- "woman in lingerie blindfolded, man holding silk ribbon, bedroom"
- "woman on top of man, riding position, hotel room, scattered clothes on floor"

### Обязательно

1. Явно указывать пол: `man`, `woman` (не "person", "partner")
2. Конкретные действия и позы
3. Место/обстановка
4. Релевантные детали (одежда, предметы)

### КРИТИЧНО: Одно действие

**Промпт описывает ОДНО конкретное действие.**

Это изображение, не видео. Нельзя показать "или то, или это". Если есть два варианта — это две разные сцены.

**Плохо:**
- "woman kneeling before man worshipping his cock, or man kneeling worshipping woman's pussy" — два варианта в одном промпте
- "man spanking woman or woman spanking man" — невозможно изобразить

**Хорошо:**
- Сцена 1: "woman kneeling before man, worshipping his cock with hands and mouth"
- Сцена 2: "man kneeling before woman, worshipping her pussy with mouth and hands"

Если действие может быть в обе стороны — создавай две сцены с разными промптами.

## title

**Назначение:** Короткое название сцены (2-4 слова).

### Примеры

- "Power Dynamic" / "Динамика власти"
- "First Time" / "Первый раз"
- "Silk Blindfold" / "Шёлковая повязка"
- "Threesome MMF" / "Секс втроём (ММЖ)"

## subtitle

**Назначение:** Опциональный подзаголовок, hook (вопрос или уточнение).

### Примеры

- "Who's in charge?" / "Кто главный?"
- "Soft or hard?" / "Нежно или жёстко?"
- "Trust and surrender" / "Доверие и отдача"

## Симметрия ролей (Role Symmetry)

**Назначение:** Избежать гендерного перекоса в сценах.

### Главное правило

**Если роль может быть у М и у Ж — нужны отдельные сцены для каждого варианта.**

Не все активности гендерно-симметричны, но многие — да. Не создавай перекос, где:
- Только Ж надевает белье/одежду
- Только Ж делает стриптиз
- Только М доминирует
- Только М активен

### Паттерны именования

Для направленных сцен используй суффиксы:

| Суффикс | Значение | Пример |
|---------|----------|--------|
| `-m-to-f` | М делает Ж | `spanking-m-to-f.json` |
| `-f-to-m` | Ж делает М | `spanking-f-to-m.json` |
| `-m-wears` | М носит/надевает | `harness-m-wears.json` |
| `-f-wears` | Ж носит/надевает | `harness-f-wears.json` |
| `-m-performs` | М исполняет | `striptease-m.json` |
| `-f-performs` | Ж исполняет | `striptease-f.json` |

### Когда нужны парные сцены

**Нужны отдельные сцены (разные image_prompt, user_description):**
- Одежда/белье (харнесс на М ≠ харнесс на Ж)
- Стриптиз (он раздевается ≠ она раздевается)
- Доминирование/подчинение (он доминирует ≠ она доминирует)
- Сервис (он обслуживает ≠ она обслуживает)
- Impact play (он шлёпает ≠ она шлёпает)

**Достаточно одной сцены с выбором внутри:**
- Позиции (mutual)
- Локации (mutual)
- Эмоциональный контекст (mutual)
- Игрушки которые используются на обоих (butt plug — есть элемент "кто носит")

### role_direction

Используй правильное значение:
- `m_to_f` — М активен, Ж пассивна
- `f_to_m` — Ж активна, М пассивен
- `mutual` — симметричная сцена
- `solo` — один участник

### Чеклист перед созданием сцены

1. Может ли эта роль быть у другого пола?
2. Если да — нужна ли парная сцена?
3. Если нужна — создай обе сразу
4. Проверь что image_prompt и user_description соответствуют конкретной роли

## Локализация

Все текстовые поля — JSONB с ключами `ru` и `en`:

```json
{
  "ru": "Русский текст",
  "en": "English text"
}
```

Переводы должны быть эквивалентны по смыслу, но естественны для каждого языка.
