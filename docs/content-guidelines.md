# Content Guidelines

> Правила написания контента для сцен: описания, промпты, заголовки.

---

## ГЛАВНЫЕ ПРИНЦИПЫ

### 1. Однозначность

**Каждая сцена = ОДНО конкретное действие.**

- Никаких "или" для разных действий
- Если действие может быть в обе стороны — две сцены
- Если действие отличается (вагинально vs анально) — две сцены

**Плохо:**
- "Он вводит руку внутрь неё — вагинально или анально" — ЭТО РАЗНЫЕ ДЕЙСТВИЯ!
- "Секс-машина трахает женщину или мужчину" — КТО КОГО?
- "Один партнёр шлёпает другого" — КТО КОГО?

**Хорошо:**
- "Он вводит руку в её влагалище" (сцена 1)
- "Он вводит руку в её анус" (сцена 2)
- "Секс-машина трахает её, он управляет" (сцена 1)
- "Секс-машина трахает его, она управляет" (сцена 2)

### 2. Второе лицо

**user_description всегда от второго лица — "ты делаешь".**

Зависит от `for_gender`:
- `for_gender: "male"` → М видит сцену → "Ты делаешь ей..."
- `for_gender: "female"` → Ж видит сцену → "Ты делаешь ему..." или "Он делает тебе..."
- `for_gender: null` → оба видят → "Ты делаешь партнёру..." или разделяй на две сцены

**Плохо:**
- "Она шлёпает его по заднице" — третье лицо
- "Мужчина и женщина занимаются сексом" — третье лицо

**Хорошо:**
- "Ты шлёпаешь её по заднице" (for_gender: male)
- "Ты шлёпаешь его по заднице" (for_gender: female)

### 3. Только факты

**Никаких эмоций, оценок, ощущений.**

Описывай ЧТО ПРОИСХОДИТ, не что чувствуют. Пользователь сам решит, нравится ему или нет.

**Плохо:**
- "Тесно, неудобно, но так возбуждающе" — откуда ты знаешь?
- "Невероятная близость и доверие" — навязываешь эмоции
- "Острые ощущения" — субъективно
- "Сладкая боль" — оценка

**Хорошо:**
- "На заднем сиденье или откинув переднее сиденье"
- "Он связывает ей руки"
- "Горячий воск капает на кожу"

## user_description

**Формула:**
```
[ТЫ] + [ЧТО ДЕЛАЕШЬ] + [КОМУ/С КЕМ] + [детали]
```

ИЛИ (если действие над тобой):
```
[ОН/ОНА] + [ЧТО ДЕЛАЕТ] + [ТЕБЕ] + [детали]
```

### Примеры по for_gender

**for_gender: "male"** (М видит):
```json
"user_description": {
  "ru": "Ты шлёпаешь её по заднице.",
  "en": "You spank her ass."
}
```

**for_gender: "female"** (Ж видит, она активна):
```json
"user_description": {
  "ru": "Ты шлёпаешь его по заднице.",
  "en": "You spank his ass."
}
```

**for_gender: "female"** (Ж видит, она пассивна):
```json
"user_description": {
  "ru": "Он шлёпает тебя по заднице.",
  "en": "He spanks your ass."
}
```

**for_gender: null** (оба видят, симметричное действие):
```json
"user_description": {
  "ru": "Ты и партнёр занимаетесь сексом в позе 69.",
  "en": "You and your partner have sex in 69 position."
}
```

## Дублирование сцен

### Запрещённые дубликаты

**Если есть направленные сцены (m-to-f, f-to-m) — generic сцена НЕ НУЖНА.**

Проверяй:
- Есть `knife-play-m-to-f` и `knife-play-f-to-m`? → Удали `knife-play`
- Есть `spanking-m-to-f` и `spanking-f-to-m`? → Удали `spanking`
- Есть `golden-shower-m-to-f` и `golden-shower-f-to-m`? → Удали `golden-shower`

Generic сцена с "один партнёр делает другому" — ВСЕГДА заменяется на две направленные.

### Когда нужно разделять

**Разные действия = разные сцены:**

| Одна сцена | Должно быть |
|------------|-------------|
| Фистинг (вагинально или анально) | fisting-vaginal-m-to-f + fisting-anal-m-to-f |
| Секс-машина (её или его) | fucking-machine-on-her + fucking-machine-on-him |
| Римминг (ему или ей) | rimming-he-to-her + rimming-she-to-him |

**Разные направления = разные сцены:**

| Одна сцена | Должно быть |
|------------|-------------|
| Шлёпки | spanking-m-to-f + spanking-f-to-m |
| Удушение | choking-m-to-f + choking-f-to-m |
| Воск | wax-m-to-f + wax-f-to-m |

## paired_scene

Связывает сцены одной темы с разными ролями.

```json
// spanking-m-to-f.json
{
  "slug": "spanking-m-to-f",
  "for_gender": "male",
  "paired_scene": "spanking-f-to-m"
}

// spanking-f-to-m.json
{
  "slug": "spanking-f-to-m",
  "for_gender": "female",
  "paired_scene": "spanking-m-to-f"
}
```

**ВАЖНО:** `paired_scene` использует HYPHEN (spanking-f-to-m), не UNDERSCORE!

## role_direction

| Значение | Кто активен | Пример |
|----------|-------------|--------|
| `m_to_f` | М делает Ж | Он шлёпает её |
| `f_to_m` | Ж делает М | Она шлёпает его |
| `mutual` | Оба равны | Поза 69 |
| `universal` | Не зависит от пола | Локация |

---

## for_gender

| Значение | Кто видит сцену | user_description от лица |
|----------|-----------------|--------------------------|
| `"male"` | Только М | "Ты делаешь ей..." |
| `"female"` | Только Ж | "Ты делаешь ему..." / "Он делает тебе..." |
| `null` | Оба | "Ты и партнёр..." |

## ai_description

Технический контекст для AI. Не показывается пользователю.

- Термины, теги
- Что тестирует сцена
- Связи с другими темами

```json
{
  "ru": "Тест на интерес к групповому сексу. MFM конфигурация.",
  "en": "Testing interest in group sex. MFM configuration."
}
```

## image_prompt

### Структура

```
[man/woman] + [действие, поза] + [место] + [детали]
```

### Правила

1. Явно указывать пол: `man`, `woman` (не "person")
2. Одно конкретное действие (это изображение, не видео)
3. Место/обстановка
4. Релевантные детали

### Запрещённые слова

```
photorealistic, realistic, photo, photography
masterpiece, best quality, high quality, detailed
4k, 8k, uhd, hd, high resolution
beautiful, stunning, amazing, perfect
natural lighting, studio lighting, dramatic lighting
```

## title / subtitle

**title:** Короткое название (2-4 слова)
**subtitle:** Hook или уточнение

```json
{
  "title": { "ru": "Шлёпки", "en": "Spanking" },
  "subtitle": { "ru": "Он шлёпает её", "en": "He spanks her" }
}
```

## Чеклист создания сцены

1. [ ] Одно конкретное действие (никаких "или")
2. [ ] Ясно КТО делает КОМУ
3. [ ] user_description во втором лице ("ты")
4. [ ] Никаких эмоций/оценок в описании
5. [ ] for_gender соответствует тому, кто видит
6. [ ] role_direction соответствует направлению
7. [ ] paired_scene указан (если есть парная)
8. [ ] Нет дублирующей generic сцены
9. [ ] image_prompt — одно действие, указан пол
